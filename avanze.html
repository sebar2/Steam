Bitácora de Avances del Proyecto

  • Fase 1: Adquisición y Plataforma Web

    15 de Agosto de 2024

    En esta etapa inicial, se sentaron las bases físicas y digitales del proyecto.

    • Adquisición de Componentes Físicos: Se realizó la compra de los componentes electrónicos esenciales para el prototipo, incluyendo el microcontrolador ESP32, el lector RFID-RC522, un servomotor y otros elementos periféricos. Esta adquisición fue un paso crucial para habilitar las futuras fases de ensamblaje y pruebas de hardware.
    • Desarrollo de la Estructura Web: Se maquetó la estructura fundamental del sitio web utilizando HTML5 y CSS3. Se empleó el framework Bootstrap 5 para asegurar un diseño responsivo y moderno. Se definieron dos páginas clave:
      • Página de Presentación (index.html): Creada para describir el proyecto, sus objetivos y características al público.
      • Panel de Administración (inicio.html): Diseñado como la base para la futura gestión del sistema de cerradura.

  • Fase 2: Dinamismo, Backend y Despliegue

    22 de Agosto de 2024

    Esta fase se centró en dar vida a la plataforma web, conectarla a un backend funcional y hacerla accesible en internet.

    • Implementación de Lógica con JavaScript: Se utilizó JavaScript (ES6 Modules) para añadir funcionalidad dinámica al panel de administración. Esto incluyó la implementación del sistema de autenticación de usuarios (registro e inicio de sesión) y la manipulación del DOM para mostrar u ocultar contenido según el estado de la sesión.
    • Configuración del Backend (Base de Datos): Se configuró un proyecto en Firebase y se integró la Realtime Database. Esta base de datos NoSQL es fundamental para la gestión remota de UIDs autorizados y para la creación de un registro de eventos en tiempo real, que muestra cada intento de acceso al sistema.
    • Despliegue en Servidor: El sitio web fue publicado en un entorno de producción. Se utilizó el servicio de hosting gratuito InfinityFree para alojar la aplicación. Los archivos del proyecto fueron transferidos al servidor mediante un cliente FTP (FileZilla), haciendo la aplicación accesible públicamente.

  • Fase 3 : Simulación y Montaje Inicial

    24 de Septiembre de 2024

    Se realizó el avance 2 del proyecto con foco en la simulación de apertura de la cerradura y la conexión del circuito para llevarlo a la práctica.

    • Resultado: El código fue verificado en simulación; luego se procedió al montaje real en protoboard.

    • Conexiones iniciales (Protoboard): ESP32E al protoboard; GND (azul) al riel negativo; 3V3 (rojo) al riel positivo.

    • Placas probadas: ESP32 Dev Module, ESP32 Espressif System, Arduino Nano ESP32.

    • Dificultad destacada: El ESP32 no calza directo en el protoboard; se usaron jumpers para enlazarlo.

    • Wokwi: Se armó circuito virtual (ESP32 + RC522 + LED + Resistor
      + Botón) y se probaron lecturas RFID, LED en D4 y botón en D15.

    • Cableado clave (Wokwi/real): LED: D4 → Resistencia 220Ω → GND. Botón: D15 ↔ GND
      (INPUT_PULLUP). RC522 (SPI): 3V3, GND, SDA→D5, RST→D22, SCK→D18, MOSI→D23, MISO→D19.

    • Nota del profesor (26 de septiembre): Se sugirió probar distintos modelos de placas ESP-32E para mejorar la compatibilidad en
      el simulador Wokwi. Ejemplos de modelos disponibles en Wokwi: ESP32 Dev Module, ESP32-WROOM-32, ESP32-PICO-D4, NodeMCU-32S
